# Infinite Canvas 设计规范

## 设计定位

Infinite Canvas 是面向 AI 内容生产的桌面工作台。视觉方向为“工业化创作工具”：强调内容密度、操作效率和画布空间，装饰应克制，层级主要依靠对比、边框、留白与排版建立。页面采用侧栏、主舞台和上下文面板组成的非对称布局；避免将所有内容放入居中的卡片网格。

本规范以 `frontend/src/index.css` 和 `frontend/src/styles/studio.css` 的现有实现为基线。新增样式必须优先复用 Token，不得在功能组件中复制一套主题。

## 字体规范

- 主字体：`Space Grotesk`，回退到 `PingFang SC`、`Microsoft YaHei` 和无衬线字体。用于导航、正文、按钮、表单和标题。
- 等宽字体：`JetBrains Mono`，用于模型 ID、路径、快捷键、参数、日志和代码；不要用于长篇正文。
- 中文强调文字仅在确有编辑语义时使用 `Noto Serif SC` / `Source Han Serif SC` / `Songti SC`。
- 字号层级：页面标题 20px；区块标题 16–19px；组件标题 13–14px；正文 12–14px；辅助信息 11–12px；标签不低于 10px。
- 正文行高使用 1.5–1.6，标题使用 1.15–1.3。常规正文权重 400–600，操作与标题 650–800；避免仅靠字重表达状态。

项目已使用 `Space Grotesk` 作为品牌字体，因此覆盖通用设计建议中对该字体的限制。除非进行全局品牌升级，不得在单个页面引入新的字体家族。

## 颜色与主题

所有主题色从 `frontend/src/index.css` 获取：

| 用途 | 亮色主题 | 暗色主题 | Token |
| --- | --- | --- | --- |
| 页面背景 | `#F7F7F8` | `#0F141D` | `--settings-bg` |
| 面板背景 | `#FFFFFF` | `#171D29` | `--settings-panel` |
| 柔和背景 | `#F1F4F8` | `#111722` | `--settings-soft` |
| 主要文字 | `#121212` | `#E5E9F0` | `--settings-text` |
| 次要文字 | `#6B7280` | `#8F9AAB` | `--settings-muted` |
| 边框 | `#E8E8EA` | `#2A3444` | `--settings-line` |

成功、警告和错误色仅用于状态反馈，分别以绿、琥珀和红为语义基础，并同时提供文字或图标，不能只依赖颜色。新增颜色前先确认现有 Token 无法表达该语义；若确需新增，应同时定义明暗主题值并更新本表。

## 间距、形状与层级

- 使用 4px 基础网格。常用间距为 4、8、12、16、24、32px；响应式外边距使用 `clamp()`。
- 交互控件建议高度为 34–40px，纯图标按钮的可点击区域不得小于 32×32px。
- 延续当前直角视觉：默认圆角为 0，不在局部组件中使用 `!important` 绕过全局规则。圆形只保留给具有明确语义的头像、状态点或滑块手柄，并需先调整全局实现策略。
- 画布连接端口（经典 `.legacy-node-port-dot`、智能 `.smart-node-port`）为小实心圆点例外：须用 `border-radius: 9999px !important` 覆盖全局直角，连线仍按端口 hit box 的 `getBoundingClientRect` 中心附着。经典节点左右端口 hit box 对称为 `-left/-right-[15px]`，使圆点落在卡片边缘内侧，避免 `overflow:hidden`（手动拉伸）时输入侧被裁切。
- 经典 API/Comfy/MS/视频生成节点：上游 PROMPT 连线文本为有效提示词（`PROMPTS · N`）；本地点击输入框在有连线时为「附加提示词」（可选，拼在连线后），无连线时为本地点主提示词。
- 经典 MODELSCOPE生成节点：模型 Tab（ZImage / Qwen Edit / Klein / 自定义）、PROMPTS、按模型显示 IMAGES 连线区、1K/比例/数量、启用 LoRA（无绑定时引导到 API → Modelscope → LoRA 管理）。
- 经典生成节点密钥门控：缺少对应 API Key / ComfyUI 实例时节点内容置灰并展示设置引导（链接到 设置 → API 或工作流），运行按钮禁用；不以静默失败代替。门控复用 `/api/config` 的 `has_ms_key` / `api_providers[].has_key`，不另建配置源。
- 默认以 1px 边框分隔区域。阴影仅用于浮层、菜单、对话框和拖拽态，常驻面板不使用厚重阴影。
- `z-index` 按内容、固定工具栏、下拉菜单、遮罩、对话框递增；不要用无依据的超大数值解决遮挡。

## 组件与交互

- 优先复用 `StudioDialog`、`StudioSelect`、`StudioTabs` 等 `frontend/src/shared/ui/` 组件；新增通用模式时先扩展共享组件。
- 图标统一使用 `lucide-react`，常规尺寸 16px，工具栏可用 18–20px。图标按钮必须提供 `aria-label` 或可见文字。
- 经典画布节点支持右下角拉伸手柄（`.legacy-node-resize-handle`）：悬停/选中时显示，最小约 220×96；手动拉伸后写入 `width`/`height` 与 `settings.sized`，小地图按实际尺寸绘制。
- 经典 OUTPUT 节点（`.legacy-output-body`）对齐 ComfyUI SaveImage：图优先、内容自适应高度；单图全宽按固有比例（无灰底 letterbox），多图密网格缩略图 `object-cover` 铺满方格；错误行紧凑置顶；未手动定尺寸时不以陈旧 `height` 撑出大片留白。
- 经典画布图片编辑弹窗（`.legacy-image-edit-*`）：近全屏舞台，预览支持滚轮/按钮缩放与拖拽平移；裁剪选区可拖移并带八向手柄；扩图为可拖图片 + 边框手柄；蒙版为画笔涂抹（无选区框），底部滑块调节笔刷直径（4–160，默认 42，对齐历史 `maskBrushSize`），悬停显示圆形笔刷预览（`.legacy-mask-brush-cursor`）。舞台使用共享透明棋盘格（`.studio-transparency-board`，灰白 16px，暗色主题对应加深），便于观察 PNG 透明区域；图片本身背景透明，不盖住棋盘。尺寸对齐素材库 Lightbox / 历史量级（面板约 `min(1480px)`，图约 `min(1300px, 100vw-100px)` × `min(840px, 100vh-200px)`）。应用裁剪/蒙版/扩图后在源节点右侧创建新 IMAGE 输入节点。OUTPUT / IMAGE 缩略图与预览弹窗预览态支持右键「创建导入图片节点」（同右侧放置、不自动连线）。
- 主按钮使用 `--settings-accent`；次按钮使用面板背景和边框；危险操作使用红色语义，并在不可逆操作前确认。
- 回收站（`TrashPanel`）：卡片多选（checkbox）+ 全选/取消全选；支持批量恢复与批量彻底删除；批量彻底删除须二次确认；操作进行中禁用批量控件并 toast 成功/部分失败计数。
- Hover 只增强已有层级，不能造成布局跳动。键盘焦点必须使用清晰的 `:focus-visible` 轮廓，不得用 `outline: none` 移除。
- 加载时间超过 300ms 时显示进度、骨架屏或加载图标；空状态说明原因并给出下一步；错误状态提供可执行的重试或修复入口。
- 动效以 140–220ms 为主，使用 `ease` 或现有缓动变量。尊重 `prefers-reduced-motion`，避免在画布拖拽、缩放等高频操作中加入装饰动画。

## 布局与响应式

- 桌面端以工作区为中心：导航保持紧凑，画布占据最大空间，属性和资源面板按任务出现。
- 素材管理页全部 5 个 Tab（图片资产、工作流管理、提示词库、画布资产、本地素材）采用统一三栏：左树 / 中列表 / 右预览。页头为「素材库管理」标题、副标题与刷新。
- 使用 Flex/Grid 和 `min-width: 0`、`min-height: 0` 控制可滚动区域，避免整页出现双重滚动条。
- 窄窗口下优先保证主任务可用：标签栏允许横向滚动，次级面板可折叠，工具栏可分组或收纳；不得仅缩小文字硬塞内容。三栏浏览器在约 900px 以下改为纵向堆叠。
- 新增界面至少检查 1280×720、1440×900 和窄窗口 900×700；同时检查中英文文案长度。

## 可访问性与验收

- 正文和关键控件遵循 WCAG AA 对比度：普通文字至少 4.5:1，大号文字至少 3:1。
- 所有功能必须可用键盘完成；对话框打开后管理焦点，关闭后将焦点还给触发元素。
- 表单控件必须有可关联标签；错误信息说明问题与解决方法。
- UI 提交前检查亮色、暗色、Hover、Focus、Disabled、Loading、Empty 和 Error 状态。视觉改动应附截图或短视频，并运行相关 Vitest；必要时补充 Playwright 测试。
